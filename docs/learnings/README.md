## 📝 ドキュメントの使い方

### 新しい学びを追加する場合
1. **実装完了後に記録する** - 計画や憶測は書かない
2. 適切なカテゴリのフォルダを選択
3. 新しいMarkdownファイルを作成
4. このREADME.mdにリンクを追加

### 記録すべき内容
- ✅ 実際に実装した技術
- ✅ 直面した課題と解決策
- ✅ 試行錯誤の過程
- ✅ パフォーマンス測定結果

### 記録すべきでない内容
- ❌ 未実装の技術の詳細
- ❌ 遭遇していない課題の想定
- ❌ 実装していない機能の説明
- ❌ 検証していないベストプラクティス

## 🎯 プロジェクト概要

### 価値提供
シェフの学びと成長を支援するマッチング/ポートフォリオプラットフォーム

### 主要機能（計画）
- スキルツリーによる学習の可視化
- 料理写真ポートフォリオ
- 「学べること」タグ付き求人マッチング
- チャット・面談予約システム
- レビュー・評価システム

### 開発体制
- 小規模チーム（FE/BE共通）
- 開発速度優先のMVP検証フェーズ
- 将来的なチーム拡大・サービス分割を見据えた設計
